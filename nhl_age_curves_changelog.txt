=========================================================
NHL AGE CURVES - CHANGELOG
=========================================================

v0.49.0 - Collapsed Controls + Forecast On by Default
------------------------------------------------------
- CHANGED: "Category & Metric" expander now loads collapsed by default to save
  vertical space; user can open it to change stats as needed.
- CHANGED: "Forecast" toggle defaults to ON so new users immediately see the KNN
  projection line when adding a skater.


v0.50.0 - Unified Player Board
------------------------------------------------------
- CHANGED: Merged separate skater_players and goalie_players session state dicts
  into a single players dict shared across Skater and Goalie modes.
- CHANGED: Adding a goalie while in Skater mode (or a skater in Goalie mode) now
  adds them to the shared board; the pipeline is_goalie gatekeeper still renders
  only the correct player type for the active category.
- CHANGED: URL param encoding updated: sk/go params replaced by a unified pl param;
  legacy sk and go params are still decoded for backward-compatible shared URLs.


v0.51.0 - Team Comparison Panel
------------------------------------------------------
- ADDED: Right-column stat panel now renders in Team mode, mirroring the player
  panel layout with one card per selected team.
- ADDED: Each team card shows the team logo (left), franchise full name and NHL
  founding year, career franchise totals (W / Pts / GF / GP), all-time wins
  rank badge in green, and best single season by wins.
- ADDED: TEAM_FOUNDED dict in nhl/constants.py with founding year for all 32 franchises.
- ADDED: get_team_all_time_stats() cached function in nhl/data_loaders.py computing
  career totals, all-time wins rank, and best season from regular-season records.
- ADDED: render_team_comparison_panel() in nhl/comparison.py.
- CHANGED: _show_panel guard in app.py now enables the 65/35 layout split for
  team mode when teams are loaded (was player-only).


v0.52.0 - Live Game Auto-Population
---------------------------------------------------------
- ADDED: App now detects the current or most recently played NHL game on first
  open and pre-loads the chart with both teams, their best skaters, and their
  starting goalies - so the chart is never empty for a new visitor.
- ADDED: nhl/schedule.py with two cached functions: get_live_or_recent_game()
  hits https://api-web.nhle.com/v1/score/now (5-minute TTL), walks back up to
  7 calendar days if today has no regular/playoff game; get_featured_players()
  fetches both team rosters, cross-references skater IDs with the historical
  parquet to pick the highest career-points skater per team, and picks the
  first-listed goalie (starters are listed first by the NHL API).
- ADDED: _default_loaded guard in app.py fires once per session after URL params
  are loaded; auto-population is skipped when a shared URL pre-populates players
  or teams so shared links are never overridden.
- CHANGED: All three chart categories are pre-populated: Skater shows the top
  scorer per team, Goalie shows the starting goalie per team, Team shows both
  franchises. User can add/remove as normal after first load.

v0.52.1 - Fix Auto-Population Endpoint
---------------------------------------------------------
- FIXED: Auto-population was silently failing because the primary score endpoint
  (score/now) times out unreliably. Switched primary to the scoreboard endpoint
  (https://api-web.nhle.com/v1/scoreboard/now) which returns ~11 days of data
  reliably in a single call with the identical gamesByDate JSON structure.
- FIXED: gamesByDate is now searched in reverse (newest-first) so the most recent
  FINAL game is found before older ones when no live game is active.

v0.52.2 - Fix Chart Key Crash on Auto-Population
---------------------------------------------------------
- FIXED: AttributeError on chart render — nhl/chart.py still referenced
  st.session_state.skater_players and st.session_state.goalie_players (removed
  in v0.50.0 unified player board). Auto-population exposed this dormant bug by
  triggering the chart on first load with players present. Changed chart cache
  key to use st.session_state.players (the unified dict).

v0.52.3 - Fix Best-Player Selection via Club Stats API
---------------------------------------------------------
- FIXED: Auto-population was picking arbitrary players (first roster entry in
  API order) instead of the team's top scorer. Root cause: parquet cross-reference
  only covers historically tracked players; current active players not in the
  parquet triggered the first-roster-player fallback silently.
- CHANGED: Replaced parquet lookup + _fetch_roster() + _find_best_skater() with
  a single call to https://api-web.nhle.com/v1/club-stats/{team}/now, which
  returns current-season points for every active player on the team (already
  split into skaters and goalies arrays). Best skater = highest points this
  season; starter goalie = most games played this season.
- REMOVED: _fetch_roster(), _find_best_skater(), load_historical_data import,
  and ROSTER_URL import from nhl/schedule.py (no longer needed).

v0.52.4 - Always-Visible Sidebar Toggle
---------------------------------------------------------
- CSS: Streamlit sidebar collapse/expand buttons now permanently visible (opacity 1)
  instead of hidden until hover -- fixes unusable toggle on touch/mobile devices.
- Minimum 36x36px tap target applied to both buttons for accessibility.


v0.52.5 - Suppress Forecast Line for Unreliable Metrics
---------------------------------------------------------
- CHANGED: Forecast projection line is now suppressed for metrics where linear
  extrapolation yields implausible results: GP (skater and goalie), SH% (skater),
  TOI (skater). Neither KNN nor the linear fallback runs for these metrics.
- Added NO_PROJECTION_METRICS set to nhl/constants.py.
- Guard added in nhl/player_pipeline.py Step 9 condition.


v0.52.6 - Fix Baseline Crash to Zero at Age 40
---------------------------------------------------------
- FIXED: 75th-percentile baseline dropped to near-zero at age 40 due to the
  centered 3-period rolling mean incorporating sparse/near-empty age-41 raw data,
  pulling the smoothed age-40 and age-39 values down before survivorship rules ran.
- Added slope-extrapolation pass in nhl/baselines.py after the survivorship loop:
  for ages 40 and 41, the baseline is floored at the value implied by continuing
  the proportional decline established by the prior two ages (slope clamped to a
  max drop of 25% per year, never allows an upward extrapolation).

v0.52.7 - Fix Baseline Age-40 Zero (Correct Location)
---------------------------------------------------------
- FIXED: Previous v0.52.6 fix targeted baselines.py but age 40 is never in the
  baseline DataFrame index (no players with 40+ GP at age 40 in the parquet), so
  the survivorship loop never fires for it.
- FIXED: Actual root cause was chart.py line 107: the age loop (range 18-41) assigns
  val=0 for any age missing from the index, which always catches age 40.
- Changed chart.py else-branch for missing ages >= 38: now extrapolates the slope
  from the prior two available ages (clamped to max 25% drop per year) instead of
  assigning 0. Age 40 baseline now continues the decline gracefully.

v0.53.0 - Fix Goalie KNN Projection (Save %, GAA, and All Goalie Metrics)
---------------------------------------------------------------------------
- FIXED: Goalie projection lines for Save %, GAA, Shutouts, Wins, and Saves
  were showing flat/straight lines instead of KNN-based curved projections.
- ROOT CAUSE 1 (Save %): Historical parquet stored Save % in 0-1 scale (0.918)
  while the live player pipeline produces Save % in 0-100 scale (91.8). The KNN
  delta update current_val += (next_avg - last_avg) applied deltas of ~0.003/yr
  to a 91.8 baseline, producing a visually flat line. Fixed by changing
  load_historical_data() in nhl/data_loaders.py: df['Save %'] = df['SavePct'] * 100.
  Also updated apply_era_to_hist() in nhl/era.py to apply the era offset in
  0-100 scale: (Save % + sv_offsets * 100).clip(0, 100).
- ROOT CAUSE 2 (GAA): The KNN extrapolation fallback (used when clone data runs
  out after age ~35) clamped pct_per_step to max(-0.08, min(0.0, ...)), preventing
  any upward projection. GAA increases (worsens) with age, so the clamp produced a
  flat line past the last clone age. Fixed in nhl/knn_engine.py: GAA now uses
  symmetric bounds max(-0.08, min(0.08, pct_per_step)).
- BONUS: Historical Save % baseline now correctly aligns with live player data on
  the chart (was invisible at 0.923 on a 87-95 scaled y-axis; now 92.3 matches).
- Clicking the projected goalie line now shows the ML Projection Clones table in
  the Season Snapshot dialog (10 position-matched historical goalies with GP, W,
  Saves, SO).
